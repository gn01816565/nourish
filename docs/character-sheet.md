# 啾啾日常 角色設定書 (Character Sheet)

> 所有美術產出（SVG / AI 生圖 / 動畫）的**單一真相之源**。
> 任何要產出新圖的人（包含 AI、Sub-Agent、外包美術）**先讀完這份**，
> 再讀 `docs/image-prompts.md`。畫面不一致是 nourish 最大的 gating risk。

---

## 0. 為什麼這份文件存在

> User feedback (2026-04-28 早)：「畫風是玩家決定要不要玩這遊戲的因素之一。」
> User feedback (2026-04-28 17:30)：「主要 TA 是女生，版面跟設計都要以女性向喜歡的為主。」

意思是：**美術不是上線後再 polish 的項目，是 MVP 上線前必須鎖死的決策**。
而且鎖的方向是**女性向審美**（粉嫩、圓潤、療癒、可愛），不是中性 / 男性向。

- 18 張 prompt 各自為政 → 玩家會看到 18 隻不同的雞 → 角色感破滅 → 養成情感投射失敗 → 留存崩盤
- **同一隻小雞**必須在「蛋 / 雛 / 幼 / 7 種終態 + 5 種心情」全部狀態下保持**可辨識的同一個體**
- 這是「**character anchor**」原則：先鎖角色，再變狀態，**不可顛倒**

## 1. 主角：啾啾（Chick the Chick）

### 1.1 一行身份卡
> 一隻**圓嘟嘟、大眼、暖黃色、橘喙**的小雞，名字「啾啾」（玩家可改），
> 個性是「期待但容易害羞」，**眼神**比體型更能識別。

### 1.2 視覺 DNA（不可動搖的 5 項）

這 5 項是**所有狀態下必須保留**的識別特徵。任何 AI 生圖如果失去其中 2 項以上，**該圖作廢重生**。

| # | 識別特徵 | 規格 | 失守判定 |
|---|---------|------|---------|
| 1 | **體色 = 暖黃 #FFD86B 主，橘 #FF9F43 點綴** | 不能是檸檬黃、芥末黃、純黃 | 偏綠或偏紅都不對 |
| 2 | **大眼，黑點 + 白色高光** | 眼睛佔頭寬 ~12%，圓形實心，左上有單一白點 | 變成杏仁眼 / 雙瞳 / 無高光都不對 |
| 3 | **小三角橘喙，朝下尖**（非鳥嘴張開狀） | 喙寬 ~14% 頭寬 | 變鸚鵡彎喙 / 全張嘴露舌都不對 |
| 4 | **圓滾身體，頭比身大或同大** | 比例：頭高 ≥ 身高 × 0.8（雛雞）/ ≥ 0.6（成雞） | 細長身體、人形比例都不對 |
| 5 | **粗黑線條 + 平塗 + 微漸層高光** | 線寬 ~3px @ 200×200 viewBox | 細線、無線、寫實渲染都不對 |

### 1.3 統一色票（不可新增硬編碼色）

```
主色 PRIMARY    #FFD86B  暖黃（雞身）
副色 SECONDARY  #FF9F43  橘（喙、爪、口腔）
口紅 ACCENT     #B23A48  暗紅（雞冠、戰鬥配件、唱歌口紅）
線稿 INK        #2C2C2C  接近黑但不純黑
紙白 BASE       #FFF8E7  奶油白（蛋殼、UI 底）
天藍 SKY        #6BCBFF  水、淚滴、智者眼鏡
草綠 LEAF       #6BCB77  蟲、草地、神祕草藥
腮紅 BLUSH      #FFB7B7  臉頰、開心特效
泥棕 EARTH      #8B5A2B  雞舍、書本、毛色變化
```

僅用以上 9 色 + 透明度。**禁用**：螢光色、霓虹色、純黑 #000、純白 #FFF。

### 1.4 構圖規則
- **viewBox**：角色與道具 `0 0 200 200`，背景 `0 0 1280 720`
- **角色置中略偏下**：cx ≈ 100, cy 中心 ≈ 110（讓頭略低於畫面中心，看起來「站著」）
- **微仰視角**：眼睛位於頭高 1/3 處（大眼 + 大頭兒童感的關鍵）
- **腳對稱**：雙腳間距 ~16-20px，每腳線條粗 4-5px

## 2. 階段變體（同一隻角色 4 種年齡）

從蛋孵到成雞是**同一個個體**長大的過程，不是換角色。AI 生圖時：

> 「想像同一隻小雞被相機在 4 個不同年齡按了快門」

### 2.1 蛋（Egg）
- **看不到雞**：橢圓殼，奶油白底 + 棕色斑點 5–7 顆
- **暗示存在**：殼型略歪向同一側（給之後雛雞的人格留個鉤子）
- **裂蛋版**：殼上有 Z 形裂縫，**從裂縫看出來的喙與一隻眼**必須符合 §1.2 規格（這是雞首次出場）

### 2.2 雛雞（Chick，2 天）
- **頭佔身體 ~50%**，身體像顆會走路的粉撲
- 翅膀只有「翅膀芽」（小弧線，不展開）
- 腳極短（10–12px）
- **眼神**：大、亮、好奇，不要太「無辜」（避免變成 mascot）

### 2.3 幼雞（Junior，2–4 天）
- 頭佔身體 ~40%，開始抽高
- 翅膀展開但小，能看出輪廓
- 雞冠**未生**或微露（紅點即可）
- 眼神：開始有個性，不再純粹好奇

### 2.4 成雞（Adult）
- 頭佔身體 ~33%
- 全翅、全冠、全尾羽
- **7 種分支見 §3**

## 3. 七種終態（同角色 + 配件 / 變色）

進化分支不是換 7 隻雞，是**啾啾長成 7 種樣子**。每個分支的關鍵：**保留 §1.2 視覺 DNA，加上 1–2 個獨特標記**。

| # | 終態 | 體色 | 標記 | 表情關鍵字 | 不可做 |
|---|------|------|------|---------|-------|
| 1 | **健康成雞 (healthy)** | 主色不變 | 紅雞冠 + 整齊羽毛紋 | 自信、安定 | 太完美無瑕 → 變得無聊 |
| 2 | **胖雞 (fatty)** | 主色不變 | 體型 ×1.4，雙腮泛粉 | 滿足、慵懶 | 醜化、抽油煙機式 |
| 3 | **醜雞 (ugly)** | 偏暗黃 #C7B370 | 毛亂、X 眼 | 委屈、迷因感 | 真醜 / 病態 / 嚇人 |
| 4 | **元氣雞 (fighter)** | 主色不變 | 粉色運動帶 + 軟拳套 | 元氣、活力 | 殺氣、肌肉誇張、暗色頭巾 |
| 5 | **智慧雞 (sage)** | 主色不變 | 半月眼鏡 + 旁邊書本 | 沉穩、內斂 | 老成 / 鬍子 |
| 6 | **歌姬雞 (diva)** | 主色不變 | 彩虹尾羽 + 麥克風 + 音符 | 燦爛、舞台感 | 性感化、年紀大 |
| 7 | **神雞 (divine)** | 偏淡黃 #FFEC8B + 光暈 | 金色光環 + 翼狀羽 + 星 | 平靜、祥和 | 攻擊性 / 太嚴肅 |

> **共通鐵律**：所有 7 種，**眼睛位置與比例不變**、**喙形不變**、**腳的橘色不變**。
> 配件可以加，但底層那隻雞要看得出是同一個個體。

## 4. 心情變體（5 種）

心情圖示是**漂浮在角色頭上的小圓 emoji**，不是替換主體。所以 5 種心情圖示的設計**獨立於角色**，但底色要用 §1.3 色票。

| 心情 | 底色 | 主圖 | 用途 |
|------|------|------|------|
| happy   | 主色 #FFD86B | ^_^ 笑眼 + 大笑嘴 + 腮紅 | mood>70 |
| neutral | 主色 #FFD86B | 圓眼 + 平嘴      | 預設 |
| sad     | 暗黃 #C7B370 | 斜眉 + 淚滴      | mood<20 / hunger<20 |
| sleeping| 主色 #FFD86B | ^^ 閉眼 + Z       | isSleeping=true |
| dirty   | 泥黃 #A88A4A | 蒼蠅 + 汗滴 + 泥點 | clean<20 |

**眼睛在心情圖示中可以變形**（笑眼、X、Z），這是 §1.2 規則的唯一例外，因為心情圖示**不是角色本體**。

## 5. 不可做（Anti-pattern Hall of Fame）

這些是 AI 最常出錯的方向。發現任何一項立刻廢圖：

1. ❌ **把雞畫成卡通鴨**（嘴變寬、扁喙）
2. ❌ **把雞畫成擬人化人形**（兩腿叉開站立 + 沒有「鳥」感）
3. ❌ **過度寫實 3D**（皮膚紋路、羽毛物理）
4. ❌ **嬰兒臉化**（眼睛 ×3 倍 + 嘴變嬰兒嘴 → 變嬰兒不是雞）
5. ❌ **每張圖體色都微微不同**（這是 AI 隨機性最常翻車的地方，必須鎖色票）
6. ❌ **配件喧賓奪主**（戰鬥雞變成 99% 拳套 + 1% 雞）
7. ❌ **背景複雜搶戲**（角色 PNG 一律透明背景，背景圖另外處理）
8. ❌ **臉的方向亂**（一張正面、一張 3/4，玩家會覺得是不同隻雞）

## 6. AI 生圖工具評估（角色一致性導向）

| 工具 | 一致性 | 控制 | 推薦度 | 備註 |
|------|--------|------|-------|------|
| **Midjourney v6 + `--cref`** | ★★★★★ | 高 | 1️⃣ | character reference 是專為角色一致性設計的 flag |
| **Stable Diffusion + LoRA** | ★★★★★ | 極高 | 2️⃣ | 需訓練 LoRA（10–20 張參考圖）。一次到位後最穩 |
| **Gemini 2.x Image Generation** | ★★★ | 中 | 3️⃣ | 可用對話式上一張當 reference，但跨 session 易漂移 |
| **GPT-image (DALL·E 3)** | ★★ | 低 | 不推 | 角色一致性弱，每次都生不同隻 |
| **Flux** | ★★★ | 中 | 備選 | 風格鎖得住，主體一致性中等 |

### 推薦工作流程（首次生圖）

1. 用 SVG 占位圖中**雛雞 (chick-baby.svg)** 截圖，當作 v0 reference
2. 在 Midjourney 跑：`/imagine [§7 全域 prompt] --cref [雛雞參考圖 URL] --cw 80`
3. 從輸出選 1 張最符合 §1.2 規格的 → 變成「**官方 v1 參考圖**」
4. 後續每張圖都用這張 v1 當 cref，僅變狀態（蛋、成雞、心情）
5. 7 個成雞分支：用 v1 雛雞當 cref，變奏年齡 + 配件

## 7. 套進每張 prompt 的全域錨定段落

```
SUBJECT IDENTITY (must apply to all variants):
A single small chick character named "Chickaboo" (啾啾). Identity is fixed:
- Body color: warm yellow #FFD86B (NEVER lemon, mustard, pure yellow)
- Eyes: large round black dots with single white highlight on upper-left
- Beak: small downward-pointing orange triangle (#FF9F43), narrow (≈14% head width)
- Body proportion: head as large as or larger than body, round and plump
- Outline: thick black ink ~3px equivalent
- Style: cute cartoon, flat shading + light gradient highlight, NO realism

FORBIDDEN: lemon yellow, duck-shaped beak, anthropomorphic human pose,
3D realistic feathers, almond-shaped eyes, double pupils.

This is the SAME individual chick across all stages and moods.
```

把這段**完整貼在每個 image prompt 的開頭**。然後再寫狀態變奏（蛋、雛、幼、7 種成雞、5 種心情）。

---

## 8. Gating Checklist（出 PNG 前 5 道關卡）

任何一張新美術出來，**逐項檢查**：

- [ ] 體色是否在 #FFD86B ± 5 範圍內（用色票工具量）
- [ ] 眼睛是否大圓 + 左上單一白高光
- [ ] 喙是否小三角朝下，不超過頭寬 14%
- [ ] 整張圖**有黑線稿**且粗細約 3px @ 200px 等比
- [ ] **跟雛雞 v1 參考圖並排看**，能不能一眼說「這是同一隻雞長大的」

5 項全過才能 commit；任 1 項不過 → 廢圖重生，**不要拼湊**。

## 10. 女性向 TA 設計約束（2026-04-28 加入）

> User 明確指定主要 TA 為女性，所有設計都要以女性向偏好為主。

### 10.1 整體調性
- **粉嫩色系優先**：粉 #FFB7B7、桃紅、暖黃；避免冷藍 / 灰 / 黑為主色塊
- **曲線 > 銳角**：所有 UI 元件圓角拉到 18-22px，按鈕橢圓化
- **甜美裝飾**：愛心 ♡、星星 ✦、花朵 🌸、緞帶、糖果 🍬 為高頻符號
- **避免**：尖刺、燃燒、爆炸、肌肉、武器、暗色金屬感

### 10.2 形象軟化清單
| 既有元素 | 軟化方向 |
|---------|---------|
| 「戰鬥雞 fighter」 | 對外文案改為「**元氣雞**」，配件由「拳套 + 暗紅頭巾」→「軟質粉色運動帶 + 軟拳套」 |
| 「打沙包」互動 | 文案改「節奏動感」感、icon 從 🥊 可考慮換 💃 或 🎯 |
| 戰鬥點數 | 對外文案 → 「活力點數」 |
| 醜雞 ugly | 維持但強化「迷因可愛」非「真醜」感 |
| 神雞 divine | 偏向「天使雞」感，光暈帶粉色調 |

### 10.3 互動 / 玩法側重
- **關係建立**（取名、對話、愛撫、想唱歌等）優先於 **競爭 / 成就堆疊**
- 截圖分享卡未來加：邊框花樣、貼紙、簽名、底色變化
- 裝扮系統（v0.3 必做）：蝴蝶結、皇冠、髮帶、花環 > 棒球帽 / 運動護具

### 10.4 文案 do / don't
- ✅ DO：「啾啾今天看起來好開心！」「想跟你撒嬌」「最喜歡主人了～」
- ❌ DON'T：「擊敗」「攻擊」「血量」「殺」「強到爆」
